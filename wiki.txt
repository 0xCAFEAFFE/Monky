'''Monky''' is a language currently under development by [[User:Menguinponky]].

It draws inspiration from [[FORTH]] and [[FALSE]], it also uses a [[Stack|data stack]] and [[Postfix_notation|reverse polish notation]] and is tailored for fast execution by a simple interpreter on a DIY 8-bit machine, while keeping the syntax somewhat readable.

* All values on the stack are 8-bit
* All keywords are single special characters
* Numeric literals are signed 8-bit integers
* ASCII chars can also be pushed onto the stack
* Input tokens are separated by spaces
* Extra whitespaces and newline characters are ignored
* Stack underflow will terminate the program
* Named variables can be used through the letters <code>a-z</code>
* Subroutines can be defined and called using letters <code>A-Z</code>
* An additional 128 byte data array is available

Note that although most instructions in the table below have familiar equivalents in their predecessor languages, there are some important differences:

The comparison operations only consume the top item and leave the previous value on the stack, also the symbols for less <code><</code> and greater <code>></code> point to the mathematically correct direction, relative to the top value.
It is often necessary to compare a value to a range of other values, for example in a <code>switch/case</code> structure, keeping the previous value on the stack easily allows that without having to duplicate it for every comparison.

Neither the instruction to print the top value as an integer <code>.</code> nor the instruction to store the top value into a variable <code>:</code> consume the top value, but the instruction to print the top as char <code>,</code> does consume it.
Chars are most likely to be printed and will therefor be popped from the stack while integers stay there for further calculation.

The logical value <code>false</code> is represented by <code>0</code> while <code>true</code> is nominally represented by <code>-1</code>, so that the <code>~</code> instruction can be used for logical inversion, though all non-zero values are interpreted as true.

The instruction character choices are supposed to make the symbols more intuitive to understand:

<code>$</code> looks like an S for <code>swap</code>

<code>^</code> is above or <code>over</code> the other tokens

<code>%</code> shows two identical elements after duplication by <code>dup</code> 

<code>_</code> symbolizes the floor that we <code>drop</code> an item to

<code>\</code> points back towards a value down in the stack to <code>pick</code>


Stack effect notation, with the top to the right:
<code>(stack before execution -- stack after execution, C syntax)</code>

Example notation:
<code>Input → Stack after execution ⇒ Output</code>

The following instructions are unique to Monky and don't affect the stack directly (n -- n):

{| class="wikitable"
! Op
! Name
! Meaning
! Example
|-
| style="text-align:center" | <code>?</code>
| style="text-align:center" | <code>skip if true</code>
| Skip next instruction if top is <code>true</code>
| <code>0 ? % → 0 0 </code>, <code>1 ? % → 1</code>
|-
| style="text-align:center" | <code>!</code>
| style="text-align:center" | <code>skip if false</code>
| Skip next instruction if top is <code>false</code>
| <code>0 ! % → 0</code>, <code>1 ! % → 1 1</code>
|-
| style="text-align:center" | <code>[</code>
| style="text-align:center" | <code>loop start</code>
| Start of a loop, <code>]</code> jumps to this symbol
| <code>[ a , ] ⇒ aaaaa... </code>
|-
| style="text-align:center" | <code>]</code>
| style="text-align:center" | <code>loop end</code>
| End of a loop, jumps back to <code>[</code> unless skipped
| <code>3 [ 1 - ! ] → 0</code>
|-
| style="text-align:center" | <code>(</code>
| style="text-align:center" | <code>block start</code>
| Start of a block, jumps to <code>)</code> unless skipped
| <code>3 ( 2 ) → 3</code>, <code>0 ? ( 2 ) → 0</code>
|-
| style="text-align:center" | <code>)</code>
| style="text-align:center" | <code>block end</code>
| End of a block, <code>(</code> jumps here
| <code>1 ? ( 2 ) → 1 2</code>
|}


== Examples ==

===[[Infinite_loop|Infinite loop]]===

<code>[ ]</code>

===[[Cat_program|Cat program]]===

<code>[ ' ! , ]</code>

===[[Looping_counter|Looping counter]]===

<code>0 [ 1 + . ]</code>

===[[Truth-machine]]===

<code>' 48 - ? ( [ . ] ) .</code>

===[[Fibonacci_sequence|Fibonacci sequence]]===

<code>0 . 1 . [ ^ ^ + . @ _ ]</code>

===[[Hello,_world!|Hello, world!]]===

<code>"hellorld" [ , ! ]</code>

===[[Comment]]===

<code>( "comments may not contain brackets" )</code>


<code>mod(12,5)</code> → <code>12 % 5 / 5 * - → 2</code>

<code>if(a==5){foo}else{bar};</code> → <code>a ; 5 = ? ( foo ) ! ( bar )</code>

<code>i=5;do{printf("%d ",i--);}while(i);</code> → <code>5 [ . 1 - ! ]</code>

Note that blocks and loops are allowed to overlap, we can for example move a string from the stack to the data array like this:

<code>"hello" -100 [ $ ^ : ? ( _ 1 + ] )</code>
