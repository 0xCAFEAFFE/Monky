'''Monky''' is yet another stack-based language designed over the course of a weekend by [[User:Menguinponky]].

== Overview ==

It draws inspiration from [[FORTH]], [[FALSE]] and [[C]], it also uses a [[Stack|data stack]] and [[Postfix_notation|reverse polish notation]] and is tailored for fast execution by a simple interpreter on a DIY 8-bit machine, while keeping the syntax somewhat readable.

* All keywords are single special characters
* All printable ASCII characters have a function
* Numeric literals are signed integers
* Single characters and strings can also be pushed onto the stack
* Input tokens are separated by spaces
* Extra whitespaces and newline characters are ignored
* Stack underflow will terminate the program
* Named variables can be used through the letters <code>a-z</code>
* Functions can be defined and called using letters <code>A-Z</code>
* An additional data array is available

== Characteristics ==

Note that although most instructions in the table below have familiar equivalents in their predecessor languages, there are some notable differences:

The comparison operations only consume the top item and leave the previous value on the stack. It is often necessary to compare a value to a range of other values, for example in a <code>switch/case</code> structure, keeping the previous value on the stack easily allows that without having to duplicate it for every comparison.

Neither the <code>print int</code> nor the <code>store var</code> instruction consume the top of the stack, instead the value stays there for further calculation. This also allows <code>print int</code> to be inserted at any point of the program for debugging output without side effects. The <code>pop char</code> instruction on the other hand prints the character and then removes it from the stack.

The instruction character choices are supposed to make the symbols more intuitive to understand:

<code>$</code> looks like the ''letter S'' that <code>swap</code> starts with

<code>^</code> is ''above'' or <code>over</code> the other tokens

<code>%</code> shows ''two circles'' after duplication by <code>dup</code> 

<code>_</code> symbolizes the ''floor'' that we <code>drop</code> an item to

<code>\</code> points ''back'' towards a value down in the stack to <code>pick</code>

<code>@</code> symbolizes the ''rotation'' of the <code>rot</code> instruction

<code>#</code> is used to <code>count</code> the ''number'' of elements on the stack

Just like in FORTH, the logical value <code>false</code> is represented by <code>0</code> while <code>true</code> is nominally represented by <code>-1</code>, so that the <code>~</code> instruction can be used for logical inversion, though all non-zero values are interpreted as true.

== Syntax ==

Stack effect notation, top of stack to the right:

<code>stack before -- stack after execution (C syntax)</code>

Example notation:

<code>Input → Stack after execution ⇒ Output</code>

| style="text-align:center" | <code>n1 n2 -- n1|n2</code>
| Bitwise OR two top items
| <code>12 10 &#124; → 14</code>
|-
| style="text-align:center" | <code>~</code>

|-

| Bitwise XOR two top items
| <code>12 6 ` → 10</code>

| style="text-align:center" | <code>=</code>
| style="text-align:center" | <code>equal</code>
| style="text-align:center" | <code>n1 n2 -- n1 n1==n2</code>

| style="text-align:center" | <code>></code>
| style="text-align:center" | <code>greater</code>
| style="text-align:center" | <code>n1 n2 -- n1 n1>n2</code>

| style="text-align:center" | <code>!</code>

| style="text-align:center" | <code>]</code>
| style="text-align:center" | <code>loop end</code>

| style="text-align:center" | <code>--</code>

| <code>{ "hello" [ % , ! ] }</code>

== Examples ==

===[[Infinite loop]]===

 [ ]

===[[Cat program]]===

 [ ' , ]

===[[Looping counter]]===

 0 [ 1 + . ]

===[[Truth-machine]]===

 ' 48 - % ? ( [ . ] ) .

===[[Fibonacci sequence]]===

 0 . 1 . [ % @ + . ]

===[[Hello, world!]]===

 "hellorld" [ , % ! ]

===[[Comment]]===

 ( "this is a comment" )

===[[Quine]]===

 "{ 1 [ \ % ? ( , 1 + ] ) _ _ } P : 34 , P ; 34 , 32 , P ;" { 1 [ \ % ? ( , 1 + ] ) _ _ } P : 34 , P ; 34 , 32 , P ;

===Other examples:===

 print(12 mod 5) → 12 5 ^ ^ / * - . → 2

 view(stack) → # [ \ . _ 1 - % ! ] _

 if(a==5){print(1)}else{print(2)} → a ; 5 = % ? ( 1 . ) ! ( 2 . )

 i=5; do{print(i--)}while(i) → 5 [ . 1 - % ! ]

 i=5; while(i){print(i--)} → 5 [ % ? ( . 1 - ] )

Note that blocks and loops are allowed to overlap, like in the last example above.

== [[Computational class]] ==

Although no definite proof exists yet, the language is assumed to be [[Turing-complete]].

== Limitations ==

Loops and blocks may not span multiple lines unless within a function.

The language does not allow recursive function calls, this is by design because these structures are rarely indispensable in practical programs, functions calling other functions are fine though.

== Interpreter ==

An interpreter is under development and can be found in a public GitHub repository: https://github.com/0xCAFEAFFE/Monky
